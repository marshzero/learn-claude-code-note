Chapter 01

Agent Loop

让LLM真正"动手"做事

问题：LLM只能给建议，不能执行

LLM 能做什么

分析问题、理解意图、给出步骤指引

LLM 不能做什么

无法真正执行任何操作——不知道当前目录有什么文件

传统方案：LLM给出步骤指引，用户手动执行，再把结果粘贴回给LLM。中间的"翻译"工作全部由人完成，LLM只是一个"顾问"。

核心问题：如何让LLM真正"动手"做事，而不是只给建议？

解决方案：用循环

答案出乎意料地简单

Agent Loop 完整流程

👤
User
提出请求
→
🧠
LLM
分析 + 决策
→
⚙️
Tool
执行操作
tool_result
↺
循环
🛑
stop_reason
控制循环退出

循环直到 stop_reason != "tool_use" 时退出

循环逻辑

  1. 用户提出请求
  2. LLM分析请求，决定是否需要调用工具
  3. 如果需要，执行工具并获得结果
  4. 把结果反馈给LLM，让它继续分析
  5. 重复直到LLM认为任务完成

核心设计：两个关键字段

stop_reason

何时停止循环？

tool_use → 继续
end_turn → 终止
max_tokens → 异常

tool_use 块

传递什么任务？

name 工具名
input 参数
id 调用ID

stop_reason — 何时停止

response = client.messages.create(
    model=MODEL, messages=messages, tools=TOOLS
)

if response.stop_reason != "tool_use":
    return  # LLM说任务完成了，退出循环

tool_use

LLM想要调用工具，循环继续

end_turn

LLM认为任务完成，循环终止

max_tokens

达到输出上限，可能是异常终止

tool_use 块 — 传递什么任务

LLM的回复是一个内容块列表，每个块可能是：

text

纯文本回复

tool_use

工具调用请求

for block in response.content:
    if block.type == "tool_use":
        tool_name = block.name
        tool_input = block.input
        tool_id = block.id

完整示例：列出Python文件

一次完整的Agent Loop数据流转

1
用户请求
{"role": "user",
 "content": "List all Python files"}
2
LLM响应 (stop_reason="tool_use")
{"stop_reason": "tool_use",
 "content": [{
   "type": "tool_use",
   "name": "bash",
   "id": "tool_123",
   "input": {"command": "ls -la *.py"}
 }]}
3
工具执行 → 返回 tool_result
{"role": "user",
 "content": [{
   "type": "tool_result",
   "tool_use_id": "tool_123",
   "content": "hello.py\ngreet.py\nmain.py"
 }]}
4
最终响应 (stop_reason="end_turn")
{"stop_reason": "end_turn",
 "content": [{
   "type": "text",
   "text": "当前目录下有3个Python文件"
 }]}

python agents/s01_agent_loop.py

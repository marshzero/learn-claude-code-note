Agent Loop

让LLM真正"动手"做事

核心思想

用简单的循环机制，让LLM能够调用工具、执行操作、获取结果，真正成为一个能"动手做事"的Agent。

while
stop_reason

问题：LLM只能给建议，不能执行

当我们让LLM"读取当前目录下的所有文件"时：

  • LLM可以分析这个问题
  • 但它无法真正执行任何操作
  • 它不知道当前目录有什么文件

传统方案：LLM给出步骤指引，用户手动执行，再把结果粘贴回给LLM。中间的"翻译"工作全部由人完成，LLM只是一个"顾问"。

核心问题：如何让LLM真正"动手"做事？

解决方案

答案出乎意料地简单

核心：while循环

while stop_reason == "tool_use":
response = llm.call(messages)

if response.stop_reason != "tool_use":
    break # 停止

execute_tools(response)
messages.append(tool_result)

stop_reason 决定是否继续循环

stop_reason：循环的开关

tool_use

LLM想调用工具
→ 继续循环

end_turn

任务完成
→ 退出循环

max_tokens

输出超限
→ 异常退出

判断逻辑
if response.stop_reason != "tool_use":
    return  # 退出循环

tool_use 块：传递任务给工具

LLM的回复是一个内容块列表，每个块可能是 text 或 tool_use

解析 tool_use 块
for block in response.content:
    if block.type == "tool_use":
        tool_name = block.name      # 工具名
        tool_input = block.input   # 参数
        tool_id = block.id          # 调用ID

messages 数组是怎么增长的？

每轮循环后，messages数组都会追加新的内容

system You are a helpful assistant. Tools: bash, read, write...
user List all Python files
assistant stop_reason: "tool_use" → {type: "tool_use", name: "bash", ...}
user tool_result: "hello.py greet.py main.py"
assistant stop_reason: "end_turn" → "目录下有3个Python文件"

messages数组记录了完整的对话历史，每一轮的请求和响应都被追加进去。

完整的闭环

1. 发请求
messages.append(user)

2. LLM思考

继续下一轮...

直到 stop_reason != "tool_use"

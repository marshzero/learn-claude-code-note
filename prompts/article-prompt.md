# 文章生成提示词

## 目标
为项目的每个章节生成"问题-解决方案"风格的文章，最终整合成一篇完整的教程。

## 文章结构模板

```
## 第一章：章节主题名

### 1. 问题引入
- 描述实际场景中的痛点
- 说明为什么这个问题需要解决
- 点出核心矛盾

### 2. 解决方案
#### 2.1 核心设计
- 解释关键技术点/设计思路
- 展示核心代码片段
- 说明为什么这样做

#### 2.2 示例数据
- 展示关键数据结构
- 展示完整的数据流转过程
- 包含请求/响应示例

### 3. 实战尝试
- 运行代码的指令
- 可尝试的任务示例
- 观察点/思考点
```

## 写作风格要求

1. **问题驱动**：每个解决方案都要回应一个具体问题
2. **代码说话**：用代码而非空话解释设计
3. **数据流转**：清晰展示信息如何流动
4. **可操作性**：提供实际可运行的示例
5. **简洁直接**：避免废话，直接切入要点
